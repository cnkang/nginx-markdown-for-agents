## Implementation Delta / Scope Change

This branch is no longer spec-only. It already includes production C/Rust
changes for runtime correctness, FFI boundary hardening, and test/runtime
validation paths. The implementation scope has expanded from pure planning to
code+validation delivery.

## Non-Goals

1. Do not achieve inclusion in Debian/Fedora/openSUSE official repositories
2. Do not claim compatibility with arbitrary NGINX vendor builds
3. Do not claim generic Kubernetes/Ingress support
4. Do not move NGINX lifecycle operations into Rust
5. Do not add Python E2E files (Rule 37)
