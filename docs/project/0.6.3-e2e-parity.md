# Rust E2E Scenario Parity

This register maps compatibility shell entrypoints to their canonical Rust
scenario implementation. A migrated wrapper contains no product assertions;
all HTTP fixtures, requests, and assertions live in `tools/e2e-harness/`.

| Compatibility entrypoint | Rust scenario | Covered behavior | Known gaps |
|---|---|---|---|
| `tools/e2e/verify_brotli_streaming_e2e.sh` | `brotli-streaming` | Real module conversion for small and large Brotli responses; single-byte upstream chunks; streaming and full-buffer routing; automatic decompression disabled; budget fail-open; malformed, truncated, and trailing input safety; slow-reader backpressure and metrics | Build-disabled linkage is verified by the Ubuntu 22.04 CI build matrix because a running binary cannot toggle compile-time support |
