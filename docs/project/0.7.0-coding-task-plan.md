## 1. Coding Task Plan Overview

This document translates the v0.7.0 spec (40 REQ IDs, 5 Workstreams, ~100 subtasks) into an
executable coding task plan with PR breakdown, per-task detail, CI/CD integration, risk mitigation,
and timeline analysis.

**Total scope**: 15 PRs, 7 Phase-0 tasks (A01-A07), 6 Phase-1 tasks (B01-B06), 6 Phase-2 tasks
(C01-C06), 5 Phase-3 tasks (D01-D05), 6 Phase-4 tasks (E01-E06), 5 Phase-5 tasks (F01-F05).

**Core principles** (inherited from AGENTS.md and agent-handoff.md):

1. Pre-output Checklist validated before every file write.
2. NGX_AGAIN is suspend-and-resume, never treat as success.
3. Body must not be sent before headers.
4. New FFI struct fields are tail-append only.
5. C side must use `*_init()` / `*_cleanup()` helpers, never literal init.
6. Delivery counter ≠ Decision counter; delivery only on downstream NGX_OK.
7. New reason code/metric/config: define → implement → test → docs → validator.
8. Coverage ≥ 80% (aggregate), ≥ 90% (critical).
9. No Python E2E files (Rule 37).
10. Shell scripts macOS bash 3.2 compatible.
