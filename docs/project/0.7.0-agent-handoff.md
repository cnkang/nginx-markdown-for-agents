## Key Constraints

### Must Follow

- All code changes comply with `AGENTS.md` Pre-output Checklist
- `NGX_AGAIN` is suspend-and-resume, **not** success
- Body must not be sent before headers
- Rust-owned memory must have FFI release functions
- New FFI struct fields are only appended to the tail (tail-append only)
- New reason codes/metrics/config must be synchronously updated: implementation + tests + docs + validators
- Coverage ≥ 80% (aggregate), ≥ 90% (critical)

### Prohibited

- Do not move NGINX lifecycle operations into Rust
- Do not hand-write literal values to initialize FFI structs on C side (use helper constructors)
- Do not collapse all decompression failures into a generic conversion_error
- Do not conflate delivery counters with decision counters
- Do not add Python E2E files (Rule 37)
- Do not claim compatibility with arbitrary NGINX vendor builds
- Do not claim generic K8s/Ingress support
