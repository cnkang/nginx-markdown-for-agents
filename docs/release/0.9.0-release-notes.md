# Release Notes: 0.9.0

**Date**: 2026-07-06
**Status**: Released
**Type**: Breaking Release (last breaking opportunity before 1.0.0 freeze)

---

## Breaking Changes

### Reason code naming convention changed to lowercase snake_case

All 18 existing reason code strings have been renamed from UPPERCASE_SNAKE_CASE to lowercase_snake_case. The count changed from 18 variants in v0.8.x to 26 variants in v0.9.0. This affects Prometheus label values, structured logs, and the diagnostics endpoint.

**Mapping (v0.8.x UPPERCASE → v0.9.0 lowercase)**:

| v0.8.x (UPPERCASE) | v0.9.0 (lowercase) |
|--------------------|--------------------|
| `CONVERTED` | `converted` |
| `DISABLED` | `disabled` |
| `NOT_ELIGIBLE` | `not_eligible` |
| `SKIPPED_ACCEPT` | `skipped_accept` |
| `SKIPPED_NO_ACCEPT` | `skipped_no_accept` |
| `SKIPPED_CONDITIONAL` | `skipped_conditional` |
| `FAILED_DECOMPRESSION` | `failed_decompression` |
| `DECOMPRESSION_BUDGET_EXCEEDED` | `decompression_budget_exceeded` |
| `DECOMPRESSION_FORMAT_ERROR` | `decompression_format_error` |
| `DECOMPRESSION_TRUNCATED_INPUT` | `decompression_truncated_input` |
| `DECOMPRESSION_IO_ERROR` | `decompression_io_error` |
| `PARSE_TIMEOUT` | `parse_timeout` |
| `PARSE_BUDGET_EXCEEDED` | `parse_budget_exceeded` |
| `REPLAY_BUFFER_ERROR` | `replay_buffer_error` |
| `SKIPPED_ACCEPT_REJECT` | `skipped_accept_reject` |
| `FFI_CALL_ERROR` | `ffi_call_error` |
| `FAILED_OPEN` | `failed_open` |
| `FAILED_CLOSED` | `failed_closed` |

**8 new reason codes added in 0.9.0:**

| Code | `as_str()` | Description |
|------|-----------|-------------|
| 18 | `conversion_error` | Generic conversion failure |
| 19 | `memory_budget_exceeded` | Memory budget exceeded |
| 20 | `overload` | Inflight guard triggered (overload protection) |
| 21 | `invalid_dynconf` | Invalid dynamic configuration |
| 22 | `degraded_snapshot` | Degraded dynconf snapshot |
| 23 | `header_plan_apply_error` | Header plan commit failure |
| 24 | `streaming_mid_flight_error` | Streaming mid-flight error |
| 25 | `bypass_no_transform` | Cache-Control no-transform bypass (conditional requests) |

These codes provide fine-grained visibility into failure modes that were previously grouped under coarser categories.

**Impact**: Any dashboard, alert rule, or log parser that matches on reason code strings must be updated. See the [Migration Guide](../guides/MIGRATION-0.9.md#observability-reason-code-naming-convention-spec-53) for the complete mapping table and regex-replace patterns.

### Metrics consolidated into unified families

Per-reason counters (e.g., `nginx_markdown_skipped_accept_total`, `nginx_markdown_parse_timeouts_total`) are consolidated into unified metric families with a `reason` label. The actual metric names in v0.9.0 are:

| Family | Purpose |
|--------|---------|
| `nginx_markdown_conversions_total` | Successful conversions |
| `nginx_markdown_skips_total` | Skipped requests (by reason) |
| `nginx_markdown_failures_total` | All errors (by reason) |
| `nginx_markdown_failopen_total` | Fail-open deliveries |

Note: There is no `nginx_markdown_failed_closed_total` metric; fail-closed rejections are tracked via `nginx_markdown_failures_total{reason="failed_closed"}`.

**Impact**: Existing PromQL queries, recording rules, and alert expressions that reference the old per-reason metric names must be rewritten to use the unified family + `reason` label pattern.

## New Features

### Diagnostics schema v1 (DRAFT)

The `/nginx-markdown/diagnostics` endpoint now returns a versioned JSON schema (`schema_version: 1`) with 6 structured sections:

- `decision` — last decision state, profile, path mode
- `inflight` — current/max/high-watermark/overload counts
- `error` — error totals, last error reason
- `streaming` — streaming eligibility and counters
- `conditional` — conditional request evaluation state
- `etag` — ETag generation policy and status

This schema is a DRAFT; it will be frozen at 1.0.0. After freeze, only additive changes (new optional fields) are permitted.

### Label whitelist enforcement

A strict label whitelist limits Prometheus metric labels to 4 allowed keys:

| Label | Purpose |
|-------|---------|
| `reason` | Decision reason code |
| `profile` | Active processing profile |
| `path_mode` | Request processing path |
| `cache_validation` | Cache validation setting |

High-cardinality labels (`url`, `path`, `user_agent`, `request_id`, etc.) are explicitly blocked to prevent unbounded time series growth.

## Technical Details

- Rust `ReasonCode` enum: 26 variants, `#[repr(u8)]`, contiguous discriminants
- FFI boundary: `markdown_reason_code_str()`, `markdown_reason_code_metric_key()`, `markdown_reason_code_count()` — all return new lowercase values
- C legacy reason strings (`ngx_http_markdown_reason.c`): removed
- Label normalization: Rust-side enforcement via `is_label_allowed()` / `is_label_blocked()` / `normalize_label_value()`
- Diagnostics schema: `components/rust-converter/src/diagnostics/schema.rs`
